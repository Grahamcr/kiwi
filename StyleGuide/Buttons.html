The Button Style Guide

Created July 13 (Friday...), 2018

Composed by Craig Graham

Key Points - Buttons:
  • We have four primary sizes of buttons: Tiny, Small, Medium & Large
  • Each application has a primary button color and a secondary button color along with three status button colors (Red, Green & Blue)
  • Every button has the same interaction styles: darker/lighter on hover, same darker/lighter color on mouse down with a 3px border.

Using consistent button design in our applications helps users recognize our call to actions and build an accurate mental model of what we expect them to accomplish in a given page/feature. We help users recognize what is clickable by making our buttons slightly rounded, using accessibility approved hover interactions and by making our buttons a consistent size and color. Each application has a primary button color (Gold/Yellow for vMOX and Dark Blue for Advantage) and a secondary button color (Light Blue for vMOX and Gold/Brown for Advantage) along with three status button colors (Red, Green & Blue). Every button has the same interaction styles, regardless of its color or size. On hover the button should turn a shade darker or lighter if the button is originally very dark (ex. Advantage Dark Blue) and on click/mouse down the button also gets a 3px border that is the original color of the button.

Our Button Sizes

We have four different button sizes, Tiny, Small, Medium and Large. As a design team we primary focus on consistency, discoverability and aesthetics when deciding which type of button to use in any given scenario. Therefore we unfortunately do not have a hard set list of rules that I can publish you let you know when one button size would be more appropriate than another.
