vMOX Standard Form

Created June 5 2018

Composed by Craig Graham

Key Points - Hea der:
  • vMOX form layout follows F-Centered Design to increase completion speed
  • The layout follows standard best practices, as documented by the UX Collective
  • Mockup can be viewed at: vMOX Standard Form Example Mockup

The vMOX Standard Form was designed to make filling out a form as painless on a user as possible. This is accomplished by following UX From best practices such as making hint text visible at all times, aligning the inputs below the label, putting options into radio buttons whenever possible instead of in dropdowns etc.

Standard Form Layout

Form Title/Header

Each form in vMOX should have a title to give the user some context around the form they will be filling out. Whenver it makes sense, it is helpful to include an icon to give further context. There are additional cases where it makes sense to include a call to action for the form at the title level, such as a "Preview" function.

Standard Form Title with Image
Input Section

vMOX forms are comprised of one or input sections. A basic input section includes a title, always-visible hint text and a input field of some type for the user to complete. The input field will vary based on the requirements of the question with some example types being, free-form text fields, text-area fields, scale selections, etc. These input sections should follow the Law of Common Region by grouping the title, hint and input close together while providing adequate and equal spacing between each input section to make for a clear distinction.

Mega Menu

The user can view the mega menu drop down at any time by clicking the menu link or the down chevron located next to the menu link. For more information on the mega menu see the Mega Menu Style Guide.

Search

When users type into the search box they should see autocomplete suggestions as well as instant results (this is not shown in the mockup). The maximum number of suggestions/results shown at a time shown at a time should be 10. To initiate the search the user can hit the enter key, click the magnifying glass icon, or choose one of the suggestions or results. The font for the search query string, suggestions and results is Roboto #333, size 14. To filter the dataset that the user is searching, they can click the dropdown to the left of the search bar and choose an option. The default filter is "All".

Header Style Details
Notifications

The notification bell is 25px by 25px, #fff. When there is a notification a 12 px by 12px #ffc627 circle appears on the lower right of the icon. The circle should be displayed until the user clicks on the notification. What happens when the user clicks on the notification icon is not yet shown in the mockup.

Settings

The user's profile picture should be displayed in a circle 25px by 25px. If the user has not uploaded a profile picture display the icon instead, fa-user-circle-o, #ffc627. To the right of the picture/icon, display "Hi," followed by the user's first name in Roboto 14px Bold, #fff. What happens when the user clicks on the settings icon is not shown yet in the mockup.

Responsive Design

A tablet version of the header mockup which has not yet been documented in this style guide, but can be viewed here: Tablet Header Bar Design Library.

Mobile

When the user is viewing this site on their mobile device there are some changes to the header. The logo, menu and notifications remain the same. The search functionality is hidden under a magnifying glass icon. When the icon is tapped the search box, filter and button are displayed in a bar under the header. They are displayed until the user clicks the magnifying glass icon in the header again. The search functionality remains the same. The settings icon and functionality remains the same but the "Hi [user's name]" is no longer displayed.
