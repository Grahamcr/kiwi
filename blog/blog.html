vMOX Backend Navigation 2.0 - Craig

Based on the results of the vMOX Navigation Card Sort Study we determined that most of our users would organize the navigation options based by role. This suggested that users would relate to a microsite layout where the user is directed to a role-based experience upon log in.

This week I began the design process by creating two A/B tests which were sent out company wide. The first A/B Test tested two designs which were purposefully design be polar opposites of one another. The first design focused on making as much of the navigation visible as possible, while the second tried to hide away as much as possible. The results of that test suggested users preferred a cleaner look, but did not want to completely hide away all of the navigation. This led to a second A/B test were one design showed all of the navigation options in a flyout menu, while the second displayed them in a contextual navigation bar.

The results of the second A/B test showed that users overwhelmingly preferred to have the navigation bar. Therefore, the second design was used to create a finalized version which has an improved look-in-feel over the version used in the A/B test. The finalized version also has a responsive view, which will adapt down to a phone with a viewport of 380px wide. If you would like to view the mockup of this design you can do so by visiting: vMOX Navigation 2.0 First Iteration Design.

Kiwi Internal Personas

Thank you to all of you who have seen the A/B Test Meg released and who gave your feedback on for the navigation system for Kiwi. Ever wonder how designers come up with the color schemes that they end up using? Meg developed the color scheme for Kiwi using a process known as an Inspiration Board which she paired with a simple set of Design Direction Principles she developed for Kiwi. You can see both in her Axure Mockup at: Kiwi Inspiration Board Mockup. The Design Direction Principles that she developed describe not only the impression that we want the entire application to give users, but also the impression the colors we use to give. For example some the principles Meg decided on for Kiwi were Simple, Relatable, To the point, Relatable and Hub. These descriptive words suggest colors that are calming, rich and closer to the blue-green spectrum, which is why you see Meg gravitate to a dark color scheme with a strong green accent color. Meg then paired that with Accessibility guidelines to develop the color scheme you interacted with in the A/B test. If anyone else would like to give their feedback on the A/B test you can view the mockup at: Kiwi A/B Test Mockup Option 1 and Kiwi A/B Test Mockup Option 2.

Data Table & Form View Kiwi Pages - Craig

This week was focused on documenting two very common views we have in all of our applications: Forms and Data Tables. Given the frequency that we encounter them I thought it would be a logical place to begin creating a deep dive Kiwi page that sets the design and interaction standards for a common design component. The style guide for the Data Table and the one for the Form do their best to not only describe the atomic components of the design and how to style them, but also to explain some of the design theory behind why I designed the page the way I did. My hope is that these style guide pages can be referenced as the source of truth for how we should display, interact with and style Data Tables and Forms across all of our applications going forward. If you get the chance I would greatly appreciate any feedback you have on these example style guides so that I better tailor the development of new pages in the future. Communications Management, A MVP Story - Craig

This week I completed the second iteration of the Communications Management Mockup for the OnePortal Action Center Effort. During the Design Review Meeting with Matt, Tim & Richard today it was decided that this part of the effort would be a great candidate for trying out a MVP approach. As a result of this decision the product team was able to narrow down the scope of this feature from a nearly 20-page mockup to a 2-page mockup. This is a great opportunity to see if focusing on smaller design/development tasks can help us achieve high quality results, better/faster alignment on direction and faster feedback loops which can help direct future efforts to focus in on user needs. The idea here is that if we can develop and release smaller bits of complete functionality and get feedback on it we can utilize that feedback to guide the decisions we make regarding what gets designed and developed next.

The MVP version of the Communications Management mockup will allow the user to see the status of each of their reports, view an example of a report and approve/deny that report for release. You can view a mockup of the new Communications Management MVP at: Communications Management MVP Mockup.

New/Interesting UX Fact (Craig)

UX is a pretty new type of job, but you can actually go back a very long ways to find its roots. UX was officially coined by Donald Norman in 1993, however you can find examples of UX further back from notable advancements made by Walt Disney, Henry Ford, Toyota & Da Vinci

UX design even finds it's way into the most unexpected places, even comedy shows. The video below is a bit by Demetri Martin about Bathroom Technology. In the joke he talks about how modern bathroom fixtures "are solutions for problems that never existed". While you are listening try to count how many UX prinicples he complains about the bathroom fixures violating.

  • Aesthetic-Usability Effect (Aesthetics over UX)
  • Lack of Visibility
  • Lack of Feedback
  • Consistency and Standards
  • Failure to meet User Expectations/Mental Model

New/Interesting UX Fact (Meg)

Since I have color on the brain this week, here are interesting UX facts about color. Did you know that Google once tested 41 shades of blue for their links to find out which color people were most likely to click on? You can read more about how to use color when designing here: Intro to Color Theory and here: Tips for choosing colors.

Also, did you know that Mark Zuckerberg is red-green colorblind? He chose blue for Facebook's primary color because it is the color that he can see best. 8% of men are red-green color blind and it is an important factor to consider when designing. You can simulate different kinds of color blindness using ColorOracle; color contrast checkers can also be very helpful.
